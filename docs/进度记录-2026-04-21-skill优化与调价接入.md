## 二、当前遇到的问题

### 2.1 smart-price-adjust 的浏览器模式不兼容

**现状**：verify-script.js 启动独立的系统 Chrome 浏览器（Playwright launchPersistentContext），需要：
- 独立的 Chrome profile 目录（automationProfileDir）
- 已登录的携程 ebooking 浏览器状态
- 系统安装的 Chrome 浏览器

**目标**：改为像爬虫一样连接 Electron 内置的 BrowserView：
- 通过 IPC 文件请求 Electron 创建隐藏 bgTab
- 通过 CDP (chromium.connectOverCDP) 连接到 Electron
- 复用 Electron session 的 Cookie（不需要独立 Chrome profile）
- 用户看不到浏览器窗口（隐藏执行）

### 2.2 改造范围评估

verify-script.js 是一个 2000+ 行的脚本，核心改动点在 `createRuntime` 函数（约 150 行）。需要：
1. 新增 `createRuntimeFromElectron` 函数，复用爬虫的 bgTab 创建 + CDP 连接机制
2. 在 `createRuntime` 入口处根据环境变量选择模式（Electron 模式 vs 独立 Chrome 模式）
3. 桥接 index.js 注入 Electron 模式的环境变量和参数
4. 处理 CDP 锁机制（和爬虫共享锁）
5. 验证 Playwright Page 对象在 CDP 模式下的行为兼容性

### 2.3 Agent 调用行为问题

从测试日志看到：
- Agent 直接用 submit_probe + submitEnabled:true（跳过 inspect_only 预览）
- 失败后用相同参数重试（不理解失败原因）
- 需要在 SKILL.md 中加强安全约束（必须先 inspect_only，用户确认后才 submit_probe）
