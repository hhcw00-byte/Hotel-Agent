# 酒店AI浏览器 - 自动化与数据持久化升级方案

## 一、系统架构设计

系统采用 Electron 桌面应用架构，左侧为浏览器视图，右侧为 AI 对话面板。AI 助手"小酒"通过自然语言理解用户意图，自动调度爬虫采集数据、查询本地数据库、生成运营建议。

### 1.1 双面板交互模型
*   **浏览器视图（左侧）：** 基于 Electron BrowserView 实现，用户可正常浏览酒店管理系统（PMS）、OTA 后台（美团、携程、飞猪等）。
*   **AI 对话面板（右侧）：** 用户通过自然语言与 AI 助手对话，AI 自动完成数据采集、分析、建议生成等工作，并实时展示执行进度和页面截图。

### 1.2 技能驱动模型
系统将所有业务能力封装为"技能"（Skill），每个技能通过 Markdown 文档（SKILL.md）定义工作流程。AI 助手根据用户意图自动选择并执行对应技能，运营人员可直接编辑 Markdown 文件来调整业务逻辑，无需修改代码。

## 二、智能爬虫

### 2.1 视觉导航
*   **核心理念：** 爬虫不依赖硬编码的 CSS 选择器，而是通过截图 + AI 视觉理解来"看懂"页面，像人一样决定点哪里、输入什么。当页面改版时无需修改代码，AI 自动适应新布局。
*   **多轮对话导航：** AI 与页面进行多轮交互——截图→分析→操作→截图→分析→操作——直到到达目标页面并提取到所需数据。
*   **智能点击：** 支持文本匹配、坐标点击等多种策略，处理下拉菜单、弹窗、折叠面板等复杂交互。

### 2.2 数据采集能力
*   **房态采集：** 从 PMS 系统获取各房型的实时入住情况、剩余库存、远期预订。
*   **后台房价采集：** 从 OTA 后台系统获取各平台的挂牌价和价格策略设置。
*   **公网价格采集：** 从携程、飞猪等公网平台获取本店及竞品的公开价格。
*   **评价采集：** 从各 OTA 平台获取客户评价内容，用于智能回复。

## 三、数据持久化与存储

为避免每次分析都重新爬取网页，系统将爬虫采集的数据持久化到本地数据库。策略类功能（如智能调价）直接从数据库读取数据，响应速度从分钟级降低到秒级。

### 3.1 存储方案
*   **结构化数据库（SQLite）：** 采用 sql.js（纯 JavaScript 实现），无需安装原生依赖，开箱即用。
*   **本地存储：** 数据库文件保存在本地，无需联网即可查询历史数据。

### 3.2 数据模型

| 数据类型 | 存储内容 | 更新方式 |
|---------|---------|---------|
| 房型信息 | 房型名称、容量 | 爬虫采集后自动写入 |
| 每日房态 | 日期、房型、挂牌价、总库存、剩余库存、状态 | 爬虫采集后按日期+房型自动更新 |
| 竞品价格 | 竞品名称、入住日期、房型、价格、是否有房、抓取时间 | 爬虫采集后追加写入 |

### 3.3 数据流转

```
爬虫采集（耗时，按需执行）        策略消费（秒级，随时可用）

  房态采集 ─┐                      智能调价
  房价采集 ─┼→ 数据持久化 → 数据库 → 竞品分析
  竞品采集 ─┘                      趋势报告
```

*   **写入侧：** 爬虫技能采集到原始数据后，通过数据持久化模块转换格式并写入数据库。支持去重（相同日期+房型不重复写入）。
*   **读取侧：** 策略技能（如智能调价）直接查询数据库获取最新数据，无需等待爬虫执行。如果数据库中数据不足，告知用户需先采集。

## 四、AI 技能体系

### 4.1 技能分类

*   **工具类技能：** 有实际执行脚本，负责与外部系统交互（爬虫、数据库操作等）。
*   **策略类技能：** 无脚本，通过 Markdown 文档定义工作流程和决策规则，AI 阅读后按步骤调用工具类技能完成任务。

### 4.2 当前技能清单

| 技能 | 类型 | 功能说明 |
|------|------|---------|
| 智能调价 | 策略 | 从数据库读取房态、房价、竞品数据，应用调价策略生成调价建议 |
| 智能回复 | 策略 | 分析客户评价内容，生成专业的回复文案 |
| 房态采集 | 策略 | 指导爬虫前往 PMS 系统采集房态数据 |
| 后台房价采集 | 策略 | 指导爬虫前往 OTA 后台采集本店房价 |
| 公网价格采集 | 策略 | 指导爬虫前往公网平台采集本店及竞品价格 |
| 竞品情报 | 策略 | 竞品市场分析 |
| 网页爬虫 | 工具 | 底层爬虫引擎，通过视觉导航访问浏览器页面并提取数据 |
| 数据库操作 | 工具 | 数据库增删改查（房型、房态、竞品价格） |
| 数据持久化 | 工具 | 将爬虫原始数据转换格式并写入数据库 |

### 4.3 技能调用机制

1.  用户输入自然语言指令（如"帮我看看今天需要调价吗"）
2.  AI 根据各技能的关键词标签自动匹配最相关的技能
3.  AI 加载该技能的完整工作流程文档
4.  AI 按文档中的步骤逐步执行（查询数据库、应用策略、生成建议）
5.  将结果以结构化格式（表格、图表）展示给用户

### 4.4 智能调价工作流

智能调价是核心策略技能，工作流程：

1.  **理解需求：** 判断用户要当天调价、短期调价还是长期价格铺设
2.  **查询数据库：** 从本地数据库获取房型列表、房态+房价数据、竞品价格数据（三个查询并行，秒级完成）
3.  **应用策略：** 根据余房量阈值、周中/周末差异、竞品对标等规则生成调价建议
4.  **输出方案：** 以表格形式展示每个房型的当前价格、建议价格、调价幅度、决策依据

## 六、性能优化

### 6.1 数据库加速策略消费
*   **问题：** 原方案每次智能调价都串行调用 3 个爬虫技能，耗时数分钟。
*   **方案：** 爬虫采集与策略消费解耦。爬虫负责定期采集数据入库，策略技能直接查数据库，响应时间从分钟级降到秒级。

### 6.2 技能加载优化
*   **元数据摘要：** 系统提示词中只包含每个技能的名称、描述和关键词标签，不加载完整内容，节省 Token 消耗。
*   **按需加载：** 仅在 AI 选定技能后才加载完整的工作流程文档。
*   **并行保护：** AI 加载技能文档时，自动跳过同批次的其他工具调用，确保先阅读完指令再执行操作。

### 6.3 截图压缩
*   爬虫截图在传输给 AI 模型前进行压缩，控制图片大小，降低 Token 消耗。
