# 数据管道重构 & 打包配置更新（2026-04-02）

## 一、背景

将酒店价格数据采集流程从"LLM Agent 驱动爬虫"重构为"直接调用 API 脚本 + Adapter 翻译 + 数据库存储"的闭环流程，并同步更新前端 Dashboard 和 Electron 打包配置。

## 二、整体闭环流程

```
心跳定时任务 (HeartbeatManager)
  │
  ├─ 1. skillManager.executeSkill('api-*', params)   // 直接调 API 脚本
  │     └─ 脚本抓取数据 → 保存到 data/api-results/
  │
  ├─ 2. skillManager.executeSkill('data-store', { source: 'api-*' })
  │     ├─ 读取 data/api-results/ 最新文件
  │     ├─ 通过 Adapter 翻译为统一格式
  │     └─ 写入 MySQL price_snapshots 表
  │
  └─ 3. 失败时 → executeAgentRecovery()
        └─ 创建 PiAgentManager → 闭环诊断修复
```

### 3.3 心跳管理器（前一轮 + 本轮修复）

文件：`src/main/heartbeat/heartbeat-manager.ts`

- 重写 `executeTask()`：直接调 API 脚本 → data-store → 失败走 Agent 恢复
- 重写 `executePlatformGroup()`：同上逻辑
- 新增 `executeAgentRecovery()`：创建临时 PiAgentManager 进行闭环修复
- **本轮 Bug 修复**：SkillExecutor 只要脚本有 stdout 就返回 `success: true`，即使脚本输出 `{ success: false }`。修复方式：
  ```typescript
  // 修复前（只检查 SkillExecutor 层面的 success）
  if (!apiResult.success) { ... }

  // 修复后（同时检查脚本输出的 success）
  if (!apiResult.success || apiResult.output?.success === false) { ... }
  ```
  `executeTask()` 和 `executePlatformGroup()` 均已修复。

## 四、打包后路径解析

| 用途 | 打包后路径 |
|------|-----------|
| 技能定义 (SKILL.md) | `app.getAppPath()/dist/skills/` (asar 内) |
| Agent 人设 | `app.getAppPath()/dist/agent/` (asar 内) |
| 脚本文件 | `resourcesPath/scripts/` (extraResources) |
| 数据文件 | `resourcesPath/data/` (extraResources) |
| 任务配置 | `resourcesPath/tasks/` (extraResources) |
| 数据库模块 | `resourcesPath/database/dist/` (extraResources) |
| mysql2 | `resourcesPath/database/node_modules/` (extraResources) |

子进程 `cwd` 设为 `getBasePath()` = `resourcesPath`，所有相对路径均基于此。

## 五、已验证的关键链路

- DB 凭据传递：主进程 `setDbEnv()` → `llmEnv` → `context.env` → 子进程环境变量 → DatabaseManager 构造函数读取
- `price_snapshots` 表在 `initialize()` 中自动创建（`CREATE TABLE IF NOT EXISTS`）
- api-runtime 仅依赖 Node.js 内置模块（http/https/url/fs/path），无第三方依赖
- data-store 的 require 路径在打包后均可正确解析

## 六、编译与打包命令

```bash
# 编译（如遇内存不足加 NODE_OPTIONS）
set NODE_OPTIONS=--max-old-space-size=8192
node scripts/prebuild          # 编译子项目
npm run build                  # webpack 编译主程序

# 数据库模块单独编译（修改 database-manager.ts 后需要）
cd database && npx tsc && cd ..

# 打包 Windows .exe 安装程序
set NODE_OPTIONS=--max-old-space-size=8192
npm run package:win            # 输出到 release/ 目录
```
