# 数据解析与 Agent 工具调用优化 — 进度文档 (2026-04-15)

## 一、本轮完成的改动

### 1. 数据解析准确性

**问题**：Agent 对话时调用 API skill 获取原始 JSON 后自己解析，经常解析错误（字段读错、数据遗漏）。

**解决方案**：
- `scripts/data-store/index.js` 新增 `parseOnly` 参数 — 传 `true` 时只走 adapter 解析不入库，返回标准化数据
- `src/main/pi-agent-manager.ts` — API skill 返回结果时自动用 adapter 解析（通过 `vm` 模块执行 adapter.js），把标准化数据替换原始 JSON 摘要返回给 Agent
- 日志标记：`[PI-AGENT-DEBUG] API-Script success (adapter parsed)` 表示走了 adapter，`(raw sample)` 表示降级到原始摘要

**关键文件**：
- `scripts/data-store/sample-extractor.js` — 从原始 JSON 提取精简数据样本（给 LLM 写 adapter 用）
- `scripts/data-store/adapter-validator.js` — 验证 adapter 输出质量

### 2. 房型名称映射

**问题**：不同平台的房型名称不一致（如 PMS 叫"豪华静谧单人间"，美团叫"豪华静谧单人间-不含早-入住当天18点前可免费取消-高速网络+连住优惠+独享空间"）。

**解决方案**：
- 管理后台"酒店信息配置"页面新增"标准房型管理"区域，用户手动配置标准房型名
- 标准名存在 `room_type_mapping` 表（`platform='canonical'`）
- 心跳入库后自动调 LLM（轻量级 HTTP 调用，不创建 Agent 实例）做跨平台映射
- Dashboard 查询通过 `LEFT JOIN room_type_mapping` 显示 `display_name`

**已知问题**：
- 名字完全不同的房型（如"温馨单人间"和"精选单人间"是同一个房间）LLM 无法自动映射，需要后续加"手动映射修正"功能
- `storeResult.output?.platform` 因 console.log 污染 stdout 导致为 undefined，已用 `SKILL_TO_PLATFORM` 映射表兜底

### 3. Agent 工具调用决策优化

**问题**：Agent 面对数据查询时决策混乱——该查数据库时去调 API，该调 API 时去调爬虫。

**解决方案**：
- `agent/persona/AGENTS.md` 精简为纯决策规则（从 6000+ 字符减到 ~1200 字符），加了数据查询优先级
- `agent/prompts/skills-usage-rules.md` 精简为 4 条核心规则
- 所有 API skill 的 SKILL.md description 加了调用场景描述（"当用户要求查看...时调用"）
- `database-operations` description 标注为"非实时查询场景"
- 所有非必填参数标注"可选"和默认值，避免 Agent 误以为需要先获取参数
- API 失效恢复段标注"仅在调用返回失败时才执行"，防止正常调用时误触发爬虫
- `api-first-instruction.md` 的 SKILL.md 生成模板同步更新，Agent 以后自己创建的 skill 也遵循这些规范

### 4. Recovery 流程增强

**改动**：
- `buildRecoveryPrompt()` 自动注入数据样本（通过 `sample-extractor`），Agent 修 adapter 时能看到真实数据结构
- `recovery.md` 新增 `{{dataSample}}` 占位符

### 5. 爬虫 API 拦截修复

**问题**：搜索建议 API 在输入框打字时触发，但爬虫到达目标后 `clearCapturedData()` + `reload()` 把导航过程中捕获的请求清掉了。另外标签页切换时 `initializePageComponents()` 重建 NetworkMonitor 也会丢失之前的捕获。

**解决方案**：
- `crawler-orchestrator.ts` — 到达目标后先保存导航过程中的请求，reload 后合并（URL 去重）
- `initializePageComponents()` — 重建 NetworkMonitor 前保存旧数据，重建后恢复
- `network-monitor.ts` — 新增 `restoreCapturedData()` 方法
- 新增"搜索建议 API 探测"机制：当目标包含搜索相关关键词时，自动逐个尝试所有文本输入框，每个框输入关键词后检查是否有新 API 被捕获

### 6. 分割线拖拽修复

**问题**：BrowserView 覆盖 resizer 导致无法拖拽。

**解决方案**：
- `split-layout.ts` — `setOnLayoutChange` 后立即同步 bounds
- mouseDown 时隐藏 BrowserView（设 bounds 为 0），mouseUp 时恢复到新位置
- 拖拽期间不更新 BrowserView bounds

### 7. 数据库查询修复

**问题**：`getPricingData` 的 `endDate` 参数没被使用，Agent 传的日期格式带时间导致拼接错误。

**解决方案**：
- SQL 改为 `BETWEEN startTime AND endTime`
- 日期参数用 `.substring(0, 10)` 只取日期部分
