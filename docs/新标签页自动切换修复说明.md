# 新标签页自动切换修复说明

## 问题描述

点击某些链接（如酒店详情链接）时会打开新标签页，但爬虫没有自动切换到新标签页，导致：
- 爬虫停留在旧页面
- 尝试多次点击同一元素（nth匹配）
- 最终导航失败

## 根本原因

1. **检测时机错误**：新标签页检测在nth匹配循环之后，而不是点击后立即检测
2. **等待时间不足**：点击后立即检查URL变化，新标签页可能还未创建
3. **检测效率低**：仅通过轮询检测，没有利用浏览器事件

## 解决方案

### 1. 添加事件监听机制
在 `BrowserController` 中添加新页面监听器：
- 监听 `context.on('page')` 事件
- 实时捕获新创建的页面
- 避免轮询延迟

### 2. 优化检测时机
在 `CrawlerOrchestrator` 中调整检测顺序：
```
点击元素 → 立即检测新标签页 → 等待渲染 → 检查URL变化 → nth匹配
```

### 3. 提升检测速度
- 轮询间隔从300ms降至100ms
- 优先检查事件监听器捕获的新页面
- 添加详细日志便于调试

## 修改文件

1. `scripts/ai-web-crawler/src/browser-controller.ts`
   - 添加 `recentNewPages` 属性
   - 添加 `setupNewPageListener()` 方法
   - 添加 `clearRecentNewPages()` 方法
   - 优化 `switchToNewTab()` 方法

2. `scripts/ai-web-crawler/src/crawler-orchestrator.ts`
   - 调整新标签页检测时机（点击后立即检测）
   - 添加 `switchedToNewTab` 标志避免重复检测
   - 优化反馈信息

## 测试建议

测试场景：
1. 点击酒店列表中的"查看详情"链接
2. 点击会打开新窗口的按钮
3. 点击target="_blank"的链接

预期结果：
- 新标签页打开后自动切换
- 日志显示检测耗时和方式
- 爬虫在新页面继续操作
