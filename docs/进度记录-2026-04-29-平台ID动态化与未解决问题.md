### 5. 心跳任务注入逻辑（src/main/heartbeat/heartbeat-manager.ts `executeTask`）

在 `executeTask` 调用 API skill 之前，新增步骤 0c：从 `hotel_config` 读取本店平台 ID 注入到 `taskParams`：

```typescript
const taskParams: Record<string, any> = { ...(task.params || {}) };
try {
  const cfg = await databaseManager.getHotelConfig();
  if (cfg) {
    if (!taskParams.hotelId && cfg.ctrip_hotel_id && task.skill === 'api-ctrip-public-price') taskParams.hotelId = cfg.ctrip_hotel_id;
    if (!taskParams.hotelId && cfg.trip_hotel_id && task.skill === 'api-trip-public-price') taskParams.hotelId = cfg.trip_hotel_id;
    if (!taskParams.hotelSlug && cfg.booking_hotel_id && task.skill === 'api-booking-public-price') taskParams.hotelSlug = cfg.booking_hotel_id;
    if (!taskParams.poiId && cfg.meituan_poi_id && task.skill === 'api-meituan-backend-price') taskParams.poiId = cfg.meituan_poi_id;
    if (!taskParams.partnerId && cfg.meituan_partner_id && task.skill === 'api-meituan-backend-price') taskParams.partnerId = cfg.meituan_partner_id;
  }
} catch (_) {}
```

然后 `this.skillManager.executeSkill(task.skill, taskParams)` 使用注入后的 params。

## 未解决的问题

### 美团后台心跳任务 poiId/partnerId 注入不生效

**现象：**
- 数据库 `hotel_config` 表中 `meituan_poi_id = '1432564683'`、`meituan_partner_id = '4919541'`，数据确认存在
- 数据库列 `meituan_poi_id VARCHAR(50)` 和 `meituan_partner_id VARCHAR(50)` 确认存在
- webpack bundle（dist/main/index.js）中注入代码确认存在且逻辑正确
- API skill 脚本（scripts/api-meituan-backend-price/index.js）确认从 params 读取，缺少时返回 MISSING_PARAM
- 但心跳执行美团后台任务时，API skill 报 MISSING_PARAM 错误，说明 poiId/partnerId 没有被注入到 params 中

**注入代码位置：** `src/main/heartbeat/heartbeat-manager.ts` `executeTask` 方法，步骤 0c

**当前代码中有一行 debug 日志：**
```typescript
console.error(`[INJECT-DEBUG] skill=${task.skill}, cfg=...`);
```
需要 rebuild 后查看这行输出来定位问题。

**可能的排查方向：**
- `getHotelConfig()` 返回的对象中是否包含 `meituan_poi_id` 字段
- `executeTask` 是否真的执行到了步骤 0c（是否被 login-check 拦截跳过了）
- taskParams 是否被后续代码覆盖
