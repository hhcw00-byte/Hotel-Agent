# 酒店AI浏览器 - 当前功能总结

## 一、核心架构

系统采用 Electron + AI Agent 架构，实现智能化的酒店业务管理。

### 1.1 基础架构
*   **前端框架:** Electron 28+ 分屏布局（左侧浏览器 + 右侧AI助手）
*   **AI引擎:** Pi Coding Agent，支持多模态大模型（Claude、GPT、Gemini）
*   **自动化引擎:** Playwright 浏览器自动化 + 视觉导航
*   **开发语言:** TypeScript + Webpack 构建

### 1.2 技能系统
*   **技能定义:** 基于 SKILL.md 的声明式技能配置
*   **技能类型:** 工具型（tool）、策略型（strategy）
*   **技能调用:** Agent 自动识别场景并调用相应技能

## 二、智能网页爬虫

### 2.1 视觉导航系统
*   **多模态理解:** 通过截图 + DOM 分析，理解页面结构和内容
*   **自动导航:** 根据目标描述自动点击、输入、滚动，到达目标页面
*   **智能重试:** 导航失败时自动尝试其他路径，避免死循环

### 2.2 数据提取策略
*   **DOM 提取:** 优先使用 DOM 结构提取数据（快速、准确）
*   **网络拦截:** 拦截 API 请求，直接获取 JSON 数据
*   **视觉识别:** 兜底方案，通过截图识别提取数据

### 2.3 页面完整性保障
*   **自动展开:** 检测并点击"展示更多"、"加载更多"按钮
*   **滚动加载:** 自动滚动触发懒加载内容
*   **日历展开:** 识别房价日历并展开完整日期范围

### 2.4 标签页管理
*   **多标签感知:** Agent 自动感知浏览器中打开的所有标签页
*   **智能切换:** 根据任务需求自动切换到目标标签页
*   **跨平台操作:** 支持在多个平台（美团、携程、飞猪）间切换

## 三、反爬虫检测规避

### 3.1 指纹保护
*   **WebDriver 移除:** 移除 navigator.webdriver 标记
*   **Canvas 噪声:** 为 Canvas 指纹添加随机噪声
*   **WebGL 保护:** 保护 WebGL 指纹不被识别

### 3.2 行为模拟
*   **人类化鼠标:** 模拟真实用户的鼠标移动轨迹（贝塞尔曲线）
*   **随机停顿:** 鼠标移动时随机停顿，更接近人类行为
*   **速度变化:** 鼠标速度随机变化，避免匀速移动

### 3.3 环境配置
*   **时区一致性:** 时区与 IP 地理位置匹配
*   **语言偏好:** 根据国家代码设置语言和地区
*   **User-Agent:** 使用真实浏览器的 User-Agent

## 四、业务场景技能

### 4.1 IM消息回复
*   **入口识别:** 自动识别页面右侧悬浮栏的 IM 图标（聊天气泡）
*   **消息提取:** 提取客户咨询内容、发送时间、客户信息
*   **回复建议:** 根据消息类型生成专业回复参考

### 4.2 竞品价格监控
*   **多平台支持:** 支持美团、携程、飞猪等主流OTA平台
*   **搜索导航:** 自动搜索竞品酒店并进入详情页
*   **价格提取:** 提取房型、价格、房态等结构化数据

### 4.3 智能调价
*   **价格分析:** 分析自身和竞品的价格走势
*   **策略建议:** 基于市场数据生成调价建议
*   **执行追踪:** 记录调价决策和市场反馈

## 五、用户交互

### 5.1 分屏布局
*   **左侧浏览器:** 完整的网页浏览功能（导航、前进、后退、刷新）
*   **右侧AI助手:** 对话式交互，支持 Markdown 渲染和代码高亮
*   **可调节分隔:** 拖动中间分隔条调整左右面板大小

### 5.2 配置管理
*   **API密钥配置:** 支持 Anthropic、OpenAI、Google、Azure、Ollama
*   **安全存储:** 配置文件加密存储在用户数据目录
*   **灵活切换:** 支持运行时切换不同的 AI 模型

### 5.3 安全设计
*   **Context Isolation:** 渲染进程与主进程完全隔离
*   **Context Bridge:** 通过安全的 API 桥接通信
*   **CSP 策略:** 内容安全策略防止 XSS 攻击

## 六、技术特性

### 6.1 测试体系
*   **单元测试:** Jest + fast-check 属性测试
*   **集成测试:** Playwright 端到端测试
*   **代码规范:** ESLint + TypeScript 严格模式

### 6.2 性能优化
*   **截图压缩:** 截图灰度化和尺寸压缩，减少 API 调用成本
*   **DOM 缓存:** 缓存页面结构，避免重复视觉分析
*   **并发控制:** 限制并发请求数，避免触发风控

### 6.3 日志系统
*   **分级日志:** 支持 debug、info、warn、error 级别
*   **结构化输出:** JSON 格式日志，便于分析和排查
*   **截图存档:** 每次操作自动保存截图，便于回溯
