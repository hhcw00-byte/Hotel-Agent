# Agent 对话演示脚本

> 按顺序演示，每个场景 1-2 轮对话，总计约 15 分钟。

---

## 一、日常对话（热身）

**1. 打招呼**
```
你：你好
预期：简短问候 + 功能引导
```

**2. 能力询问**
```
你：你能帮我做什么？
预期：列出核心能力（查数据/调价/竞品/回复/报告/采数据）
```

---

## 二、数据库价格查询

**3. 查当天房价**
```
你：今天房价多少？
预期：调用 database-operations → 返回各房型各渠道价格 + 数据采集时间
```

**4. 查入住率**
```
你：这周入住率怎么样？
预期：查本周数据 → 表格展示每日入住率/ADR/RevPAR + 采集时间
```

**5. 渠道对比**
```
你：携程和美团哪个价格低？
预期：按平台分组对比 → 表格展示差异
```

---

## 三、竞品模块

**8. 竞品价格对比**

```
你：我们跟竞品比价格怎么样？
预期：有竞品数据 → 本店vs竞品表格 + 定位分析
      无竞品数据 → 明确说"竞品数据暂无"，不编造数字
```

---

## 四、智能调价

**10. 模糊调价需求**
```
你：帮我调个价
预期：反问时间段和平台（不猜测）
```

**11. 明确调价**

```
你：今天的的，携程
预期：查数据 → 调价建议表格 + 依据
      如果无竞品数据 → 声明"⚠️ 未参考竞品数据"
      等用户确认后才执行
```

---

## 六、评价回复

**12. 写评价回复（需提前打开评价页面）**

```
你：帮我写回复
预期：提取当前页面评价 → 生成回复参考（引用格式）
      如果页面不对 → 提示切换到评价页
```

---

## 七、异常场景

**13. 数据获取失败（模拟：不登录某平台直接查）**

```
你：帮我看看携程实时价格
预期（未登录时）：引导登录，不说技术术语（HTTP/Cookie/Token）
```

**15. 数据为空**

```
你：下个月的房价多少？
预期（远期无数据时）：直说"该时段暂无数据"，不编造
```

**17. 超出能力**

```
你：酒店装修预算怎么做？
预期：坦诚说超出能力范围，建议联系工程部
```

---

## 演示要点提醒

- 每次数据展示必须带**采集时间**
- 竞品数据缺失时必须**声明未参考竞品**
- 异常场景不出现任何技术术语
- 数据缺失不编造，直接说"暂无"
- AI 回复为**流式输出**，文字逐步显示
