# Agent 标签页管理功能

## 功能概述

Agent 现在可以自主感知和管理浏览器标签页，包括：
- 查看当前打开的所有标签页
- 切换到指定的标签页
- 在多个标签页之间智能导航

## 实现原理

### 1. 标签页上下文注入

Agent 的系统提示词中会自动注入当前浏览器的标签页信息：

```
## 浏览器标签页上下文

当前浏览器中打开了以下标签页：

- **美团酒店管理系统** ✓ [当前活跃]
  - 标签页ID: tab-1
  - URL: https://pms.meituan.com/dashboard

- **携程商家后台**
  - 标签页ID: tab-2
  - URL: https://ebooking.ctrip.com/

**重要提示**：
- 你可以使用 `ai-web-crawler` 技能的 `list_tabs` 操作来获取最新的标签页列表
- 你可以使用 `ai-web-crawler` 技能的 `switch_tab` 操作来切换到指定的标签页
- 当用户提到"切换到XX标签页"、"打开XX页面"时，你应该主动判断是否需要切换标签页
- 如果目标内容可能在其他标签页中，你应该主动切换过去查看
```

### 2. 爬虫技能增强

`ai-web-crawler` 技能新增了两个操作：

#### list_tabs 操作

列出所有打开的标签页：

```json
{
  "operation": "list_tabs"
}
```

返回：
```json
{
  "tabs": [
    {
      "id": "tab-1",
      "title": "美团酒店管理系统",
      "url": "https://pms.meituan.com/dashboard",
      "active": true
    },
    {
      "id": "tab-2",
      "title": "携程商家后台",
      "url": "https://ebooking.ctrip.com/",
      "active": false
    }
  ]
}
```

#### switch_tab 操作

切换到指定的标签页：

```json
{
  "operation": "switch_tab",
  "tab_keyword": "美团"
}
```

返回：
```json
{
  "message": "已切换到标签页: 美团",
  "currentUrl": "https://pms.meituan.com/dashboard",
  "tabs": [...]
}
```

## 使用场景

### 场景 1：用户询问当前页面

**用户**："当前打开了哪些页面？"

**Agent 行为**：
1. 调用 `ai-web-crawler` 的 `list_tabs` 操作
2. 返回所有标签页的列表

### 场景 2：用户要求切换标签页

**用户**："切换到携程页面"

**Agent 行为**：
1. 先调用 `list_tabs` 查看当前标签页
2. 识别出携程对应的标签页
3. 调用 `switch_tab` 切换到携程标签页
4. 确认切换成功

### 场景 3：多平台数据采集

**用户**："帮我看看美团和携程的今日订单"

**Agent 行为**：
1. 调用 `list_tabs` 查看当前标签页
2. 调用 `switch_tab` 切换到美团标签页
3. 调用 `fetch_data` 提取美团订单数据
4. 调用 `switch_tab` 切换到携程标签页
5. 调用 `fetch_data` 提取携程订单数据
6. 汇总并返回结果

### 场景 4：新标签页处理

**用户**："点击 IM 图标回复消息"

**Agent 行为**：
1. 在当前页面点击 IM 图标（可能会打开新标签页）
2. 调用 `list_tabs` 查看是否有新标签页打开
3. 如果有新标签页，调用 `switch_tab` 切换到新标签页
4. 在新标签页中继续操作

## 注意事项

### 1. 标签页匹配规则

`tab_keyword` 会同时匹配标签页的标题和 URL（不区分大小写），匹配到第一个包含该关键词的标签页。

**重要**：当同一平台有多个系统时（如美团 OTA 后台和 PMS 系统），应使用 URL 片段来精确区分：
- 美团 OTA 后台：`me.meituan.com`
- 美团 PMS 系统：`pms.meituan.com`

### 2. Agent 决策建议

Agent 应该在以下情况下主动使用标签页管理功能：
- 用户明确提到"切换"、"打开"、"查看XX页面"
- 需要在多个平台之间切换时
- 点击链接后可能打开新标签页时
- 目标内容可能在其他标签页中时

### 3. 性能考虑

- `list_tabs` 操作非常快速，可以频繁调用
- `switch_tab` 操作会触发浏览器标签页切换，有轻微延迟
- 建议在切换标签页前先调用 `list_tabs` 确认目标标签页存在

## 测试建议

### 测试用例 1：查看标签页列表

1. 打开多个标签页（美团、携程、飞猪等）
2. 向 Agent 询问："当前打开了哪些页面？"
3. 验证 Agent 能正确列出所有标签页

### 测试用例 2：切换标签页

1. 打开多个标签页
2. 向 Agent 说："切换到携程页面"
3. 验证浏览器确实切换到了携程标签页
4. 验证 Agent 返回了切换成功的确认信息

### 测试用例 3：多平台数据采集

1. 打开美团和携程标签页
2. 向 Agent 说："帮我看看美团和携程的今日订单"
3. 验证 Agent 能自动切换标签页并提取数据
4. 验证返回的数据包含两个平台的信息

### 测试用例 4：新标签页处理

1. 在某个页面点击会打开新标签页的链接
2. 验证 Agent 能检测到新标签页
3. 验证 Agent 能自动切换到新标签页继续操作

## 未来改进方向

1. **标签页持久化**：支持保存和恢复标签页会话
2. **标签页分组**：支持按平台或功能对标签页分组
3. **标签页历史**：记录标签页切换历史，支持"返回上一个标签页"
4. **智能预测**：根据用户意图预测需要切换的标签页
5. **并行操作**：支持在多个标签页中并行执行操作
