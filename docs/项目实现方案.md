# 酒店AI浏览器 - 产品技术实现方案

## 一、项目概述

### 1.1 产品定位
酒店行业智能运营助手，基于Electron的AI浏览器应用，集成智能爬虫、AI对话、自动化操作等能力，帮助酒店运营人员提升工作效率。

### 1.2 核心价值
- **智能数据采集**: 自动获取房态、房价、竞品、评价等数据
- **AI辅助决策**: 基于数据分析提供调价、回复等建议
- **自动化操作**: 减少重复性人工操作
- **知识记忆**: 记住酒店信息，避免重复配置

### 1.3 技术架构
```

│                    Electron 应用                         │
```

## 二、核心技术模块

### 2.1 AI智能爬虫系统

#### 技术选型
- **浏览器自动化**: Playwright + CDP (Chrome DevTools Protocol)
- **视觉理解**: Gemini/Claude 多模态大模型
- **通信协议**: CDP远程调试 (端口9222)

#### 核心能力

**1. 视觉导航**
- 截图页面 → LLM分析 → 理解页面结构 → 决策下一步操作
- 无需维护CSS选择器，自适应页面变化
- 支持复杂的多步导航流程

**2. 智能元素定位**
- DOM结构分析：提取可点击元素（菜单、按钮、链接）
- 文本匹配：通过元素文本内容定位
- 选择器匹配：处理图标等无文本元素
- 浮动元素识别：右侧悬浮栏、IM图标等

**3. 多标签页管理**
- 列出所有打开的标签页
- 按关键词切换标签页（匹配标题或URL）
- 检测新标签页打开（如IM窗口）
- 多标签页场景强制要求指定目标

**4. 输入支持**
- 识别输入框（placeholder匹配）
- 自动填充文本
- 支持搜索场景
- 多输入框重试机制

**5. 数据提取策略**
- **DOM提取**: 结构化HTML解析
- **网络拦截**: 捕获API请求返回的JSON
- **视觉识别**: LLM从截图中提取信息
- **三重融合**: 综合多种策略提高准确率

**6. 页面完整性保障**
- 自动检测"展示更多"、"加载更多"按钮
- 自动展开折叠内容
- 确保数据完整性

#### 关键技术点

**菜单展开检测**
- 对比点击前后的DOM状态
- 识别新出现的子菜单项
- 避免重复点击父菜单
- 给LLM提供详细的菜单变化反馈

**SPA路由识别**
- 检测URL未变但内容更新的情况
- 对比页面主内容变化
- 判断是否成功导航

**预导航检查**
- 避免LLM重复调用导致的冗余导航
- 先判断是否已在目标页面
- 提高执行效率

**搜索场景处理**
- 识别多个输入框（目的地 vs 酒店名称）
- 优先级选择正确的输入框
- 失败后自动尝试其他输入框

### 2.2 反爬虫检测规避系统

#### 设计目标
- 隐藏浏览器自动化特征
- 模拟真实用户行为
- 保护浏览器指纹一致性
- 确保网络请求正常

#### 技术实现

**1. 指纹保护**
- **WebDriver标记移除**: 删除navigator.webdriver属性
- **Canvas指纹保护**: 注入微小噪声，每次略有不同
- **WebGL指纹保护**: 随机化渲染器参数
- **Chrome对象模拟**: 创建window.chrome及其属性
- **Plugins覆盖**: 返回非空插件列表

**2. 行为模拟**
- **贝塞尔曲线鼠标轨迹**: 生成平滑自然的移动路径
- **随机速度变化**: 模拟人类操作的不均匀性
- **随机停顿**: 按概率添加短暂停顿
- **人类化点击**: 按下和释放之间的随机延迟

**3. 网络保护**
- **Puppeteer Stealth插件**: 自动处理多种检测点
- **真实User-Agent**: 根据平台生成匹配的UA
- **标准HTTP Headers**: Accept、Accept-Language等
- **TLS/HTTP2指纹**: 确保与真实浏览器一致

**4. 环境配置**
- **时区匹配**: 根据国家代码设置对应时区
- **语言偏好**: 与时区匹配的语言设置
- **真实视口**: 从常见分辨率中随机选择
- **地理位置**: 可选的经纬度配置

**5. 启动参数优化**
- 禁用自动化控制特征
- 启用网络服务特性
- 禁用渲染器代码完整性检查

#### 验证方法
- bot.sannysoft.com - 综合检测
- fingerprintjs.com - 指纹一致性
- browserleaks.com - 多维度检测

### 2.3 Agent记忆系统

#### 系统架构
- **分层设计**: API层 → 管理层 → 检索层 → 序列化层 → 存储层
- **进程隔离**: 主进程负责存储，渲染进程通过IPC访问
- **数据格式**: JSON本地文件存储

#### 核心功能

**1. 酒店实体管理**
- 存储本店和竞品酒店信息
- 记录酒店类型（own_hotel / competitor）
- 关联多个OTA平台信息（携程、飞猪、美团等）
- 记录创建时间、更新时间、使用频次

**2. 智能检索**
- 按类型过滤（本店/竞品）
- 按平台过滤
- 模糊名称搜索
- 组合条件查询
- 多维度排序

**3. 数据持久化**
- JSON格式存储在用户本地目录
- 自动备份机制
- 数据损坏恢复
- 原子性写入保证

**4. Skill集成**
- 为调价、竞品查询等Skill提供数据
- 自动学习：爬虫执行时自动提取酒店信息
- 降级处理：记忆系统不可用时提示用户手动输入

**5. IPC通信**
- 渲染进程通过IPC调用
- 主进程处理并返回结果
- 事件通知机制

#### 数据模型
- **HotelEntity**: 酒店实体（ID、名称、类型、平台信息、时间戳等）
- **PlatformInfo**: 平台关联（平台名称、酒店ID、URL、验证时间）
- **MemoryQuery**: 查询条件（类型、平台、关键词、排序等）

### 2.4 Skill系统

#### Skill类型

**1. Tool Skill (工具型)**
- 执行具体操作，返回结构化数据
- 示例：ai-web-crawler（智能爬虫）

**2. Strategy Skill (策略型)**
- 编排多个工具，提供决策建议
- 示例：smart-pricing（智能调价）、competitor-pricing（竞品定价）

#### 核心Skill

**ai-web-crawler (智能爬虫工具)**
- 操作类型：fetch_data（导航提取）、list_tabs（列出标签页）、switch_tab（切换标签页）
- 参数：tab_keyword、target、extraction_goal、navigation_hint、max_steps等
- 返回：结构化数据、导航路径、截图、置信度

**competitor-pricing (竞品定价策略)**
- 从记忆系统获取竞品列表
- 查询外网平台价格（携程、飞猪等）
- 多平台价格对比
- 竞争态势分析
- 降级处理：记忆系统无数据时提示用户

**smart-pricing (智能调价策略)**
- 串行收集三类数据：房态 → 本店房价 → 竞品房价
- 应用调价策略（当天/短期/长期）
- 生成调价方案（类型、幅度、房型、时机、依据）
- 风险评估和执行建议

**room-pricing (本店房价查询)**
- 查询PMS系统定价
- 查询各OTA平台挂牌价
- 价格一致性分析

**room-status (房态查询)**
- 当前房态统计（OC/OD/VC/VD）
- 近期房态分析（未来3-7天）
- 远期房态日历（未来1个月）

**review-reply (评价回复)**
- 获取未回复的客户评价
- 情感分析
- 生成回复文案
- 支持用户修改

**im-reply (IM消息处理)**
- 识别IM入口（左侧菜单或右侧悬浮图标）
- 处理新标签页打开
- 提取未读消息
- 消息内容分析

#### Skill执行机制
- 子进程执行，隔离环境
- 进度事件回调
- 超时控制
- 错误处理和重试

### 2.5 Electron应用架构

#### 进程模型

**Main Process (主进程)**
- 应用生命周期管理
- 窗口和BrowserView管理
- IPC通信处理
- 文件系统访问
- 记忆系统
- Skill管理
- 配置管理
- 日志系统

**Renderer Process (渲染进程)**
- 用户界面渲染
- AI对话面板
- 网页浏览控制
- 通过IPC调用主进程功能

**Preload Script (预加载脚本)**
- Context Bridge安全桥接
- 暴露受限API给渲染进程

#### 安全设计
- Context Isolation: 启用上下文隔离
- Node Integration: 禁用Node集成
- Web Security: 启用Web安全
- Content Security Policy: 配置CSP
- API密钥安全存储

#### 窗口布局
- **分屏设计**: 左侧BrowserView（网页浏览）+ 右侧HTML（AI对话）
- **可调整**: 拖动分隔条调整左右比例
- **标签页管理**: 支持多标签页切换

#### CDP连接
- Electron开启远程调试端口9222
- 爬虫通过CDP连接到浏览器
- 共享浏览器实例，避免重复登录

## 三、业务流程

### 3.1 智能调价流程
1. 用户发起调价请求
2. AI助手调用smart-pricing策略
3. 串行收集数据：
   - room-status获取房态
   - room-pricing获取本店房价
   - competitor-pricing获取竞品房价（从记忆系统获取竞品列表）
4. 应用调价策略（基于房量、房价、竞品）
5. 生成调价方案（上调/下调/维持）
6. 展示决策依据和风险提示
7. 用户确认后执行

### 3.2 竞品价格查询流程
1. 用户请求查看竞品价格
2. competitor-pricing从记忆系统获取竞品列表
3. 如无竞品，提示用户配置
4. 对每个竞品：
   - list_tabs查看可用标签页
   - switch_tab切换到目标平台（如携程）
   - fetch_data导航到酒店详情页并提取价格
5. 汇总分析竞品价格
6. 展示对比表格

### 3.3 评价回复流程
1. 用户请求回复评价
2. review-reply调用爬虫
3. 导航到评价管理页面
4. 提取未回复的评价列表
5. 对每条评价：
   - 情感分析
   - 生成回复文案
   - 展示给用户
   - 用户确认或修改
6. 填写回复并提交

### 3.4 IM消息处理流程
1. 用户请求查看IM消息
2. im-reply调用爬虫
3. 识别IM入口（左侧菜单或右侧悬浮图标）
4. 点击IM入口
5. 如打开新标签页，自动切换
6. 提取未读消息
7. 分析消息内容
8. 展示消息摘要

## 四、关键技术点

### 4.1 多标签页管理
- **问题**: 爬虫需要在多个标签页间切换操作
- **方案**: 
  - Electron维护标签页列表
  - 爬虫通过文件IPC请求切换
  - Electron切换后爬虫更新page引用
  - 多标签页场景强制要求指定tab_keyword

### 4.2 菜单展开检测
- **问题**: 点击父菜单后子菜单展开，但URL不变
- **方案**:
  - 捕获点击前后的DOM状态
  - 对比可见菜单项列表
  - 检测新出现的子菜单
  - 给LLM提供详细反馈，避免重复点击父菜单

### 4.3 输入框处理
- **问题**: 搜索场景需要识别和填充输入框
- **方案**:
  - 提取所有输入框（placeholder匹配）
  - LLM决策输入内容和目标输入框
  - 自动填充并触发搜索
  - 多输入框重试机制

### 4.4 SPA路由检测
- **问题**: 单页应用URL不变但内容更新
- **方案**:
  - 对比点击前后的主内容
  - 检测DOM变化
  - 给LLM提供明确反馈

### 4.5 预导航检查
- **问题**: LLM多次调用fetch_data导致重复导航
- **方案**:
  - 在导航前先检查是否已在目标页面
  - 截图+DOM提示给LLM判断
  - 如已到达则跳过导航

### 4.6 反检测集成
- **问题**: 自动化特征导致账号被封
- **方案**:
  - Electron启动参数禁用自动化标记
  - 页面加载前注入保护脚本
  - 使用Stealth插件
  - 人类化鼠标轨迹

### 4.7 数据提取融合
- **问题**: 单一策略可能不准确
- **方案**:
  - DOM提取：结构化但可能不完整
  - 网络拦截：准确但不是所有数据都有API
  - 视觉识别：兜底但可能有误差
  - 三重融合提高准确率

### 4.8 记忆系统集成
- **问题**: 用户每次都要指定竞品酒店
- **方案**:
  - 记忆系统存储竞品列表
  - Skill自动从记忆系统获取
  - 降级处理：记忆系统不可用时提示用户
  - 自动学习：爬虫执行时自动提取酒店信息
