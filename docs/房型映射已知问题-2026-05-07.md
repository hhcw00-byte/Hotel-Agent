# 房型映射已知问题

> 日期：2026-05-07

## 问题描述

房型映射功能基本生效（携程后台数据已正确映射），但 Trip.com 数据存在映射错误：
- 7 条房型数据只显示了 3 条
- 显示的价格与实际数据不对应

## 根因分析

当前匹配逻辑是**包含匹配**（INSTR），当多条映射的值存在子串包含关系时，会产生交叉匹配。

### 具体案例

Trip.com 实际采集到的 room_name（来自 physicRoomMap.name）：

| # | 实际 room_name |
|---|---|
| 1 | Selected Single Room |
| 2 | Discount Single Room |
| 3 | Bed In Deluxe Hotel Room - Single Occupancy (Shared Bathroom) |
| 4 | Bed In Guestroom - Double Occupancy (Desk) (Females Only, Shared Bathroom, Room Rate Calculated Per Person) |
| 5 | Bed In Guestroom - Double Occupancy (Desk) (Males Only, Shared Bathroom, Room Rate Calculated Per Person) |
| 6 | Bed In Guestroom - Triple Occupancy (Desk) (Males Only, Shared Bathroom, Room Rate Calculated Per Person) |
| 7 | Male Quadruple Room [Shared Bathroom + Private Wardrobe + Desk] (Dormitory) |

| 男生双人间 | Bed In Guestroom - Double Occupancy |
| 男生三人间 | Bed In Guestroom - Triple Occupancy ( |
| 豪华单人间 | Bed In Deluxe Hotel Room - Single Oc... |
| 特惠单人间 | Discount Single Room<room only> |
| 精选单人间 | Selected Single Room(Room rate calcu... |
| 男生四人间 | Male Quadruple Room [Shared Bathro... |

### 冲突点

1. **女生双人间和男生双人间的映射值相同**：都填了 "Bed In Guestroom - Double Occupancy"，无法区分 Females Only 和 Males Only
2. **包含匹配可能交叉**：比如 "Single Room" 子串同时出现在多个不同房型名中

## 影响

- 部分房型数据被错误映射到其他统一名称
- 部分房型数据因匹配冲突未显示
- 价格显示错误（A 房型的价格显示在 B 房型名下）

## 临时解决方案（用户侧）

用户在映射表中填写更精确的值，包含足够的区分信息：

| 统一名称 | 应填写的 Trip 映射值 |
|---------|---------------------|
| 女生双人间 | Bed In Guestroom - Double Occupancy (Desk) (Females |
| 男生双人间 | Bed In Guestroom - Double Occupancy (Desk) (Males |
| 男生三人间 | Bed In Guestroom - Triple Occupancy |
| 豪华单人间 | Bed In Deluxe Hotel Room |
| 特惠单人间 | Discount Single Room |
| 精选单人间 | Selected Single Room |
| 男生四人间 | Male Quadruple Room |

## 待优化方向

1. **匹配算法优化**：当一条数据匹配到多个映射时，应该取匹配度最高的（而非仅按长度排序）。可考虑：
   - 精确匹配 > 前缀匹配 > 包含匹配
   - 匹配覆盖率（匹配字符数 / 总字符数）作为权重
   
2. **冲突检测**：保存映射时检测是否有多条映射会匹配到同一条数据，提示用户修改

3. **数据预览**：在映射配置 UI 中展示已采集到的实际 room_name 列表，辅助用户填写
