# 一键调价 Electron bgTab 模式集成文档

> 更新日期：2026-04-21

## 一、背景

smart-price-adjust 原本是一个独立项目（`smart-price-adjust/`），通过 `launchPersistentContext` 启动独立 Chrome 窗口，登录携程 ebooking 后台，自动化操作批量改价页面。

现在需要将它集成到 Hotel AI Browser 项目中，复用 Electron 左侧浏览器（已登录的 BrowserView），通过 CDP 连接 + bgTab 模式执行调价，不再需要独立 Chrome 窗口。

## 二、集成架构

### 原独立模式
```
用户运行 node cli.js
  → host-adapter.js → core.js → router.js → ctrip/executor.js
    → ctrip/script-adapter.js → ctrip/verify.js (execFile)
      → ctrip/verify-script.js
        → chromium.launchPersistentContext() 启动独立 Chrome
        → 操作页面改价
```

### 集成后 Electron 模式
```
Agent 调用 smart-price-adjust skill
  → SkillExecutor spawn scripts/smart-price-adjust/index.js
    → index.js 判断 Electron 模式（SKILL_NAME 环境变量存在）
    → runElectronModeDirect() 直接 spawn verify-script.js（跳过 host-adapter 链路）
      → verify-script.js
        → createRuntimeFromElectronBgTab()
          → 获取 CDP 锁（与爬虫共享）
          → chromium.connectOverCDP("http://localhost:9222") 连接 Electron
          → IPC 文件请求 Electron 创建 bgTab（隐藏 BrowserView）
          → 在 bgTab 中操作页面改价
          → cleanup: 释放 CDP 锁 + 销毁 bgTab
```

### 为什么跳过 host-adapter 链路？

原链路 `index.js → host-adapter → ... → verify.js(execFile) → verify-script.js` 有两层 execFile 嵌套（SkillExecutor spawn 是第一层，verify.js execFile 是第二层）。在这个环境下 Playwright 的 `connectOverCDP` 会超时。直接 spawn verify-script.js 消除了嵌套，和爬虫的架构一致。

非 Electron 模式（独立 Chrome）仍然走原链路，不受影响。

## 四、解决的 Bug（共 7 个）

### Bug 1：connectOverCDP 超时（恶性循环）

- **根因**：残留 bgTab 加载重页面（ebooking.ctrip.com）时，Playwright 的 CDP 初始化阶段枚举 targets 被阻塞。第一次运行创建 bgTab → 异常退出 bgTab 残留 → 下一次 connectOverCDP 永远超时。
- **修复**：index.js 直接 spawn verify-script.js 绕过 execFile 嵌套；connectOverCDP 失败时 IPC 清理 bgTab 再重试。

### Bug 2：verify-script.js 进程不退出

- **根因**：`lockKeepAlive` 的 `setInterval` 在成功路径上没被清除，阻止 Node.js 事件循环退出。
- **修复**：`createRuntimeFromElectronBgTab` 返回 cleanup 函数，finally 块中调用。

### Bug 3：stdout 未 flush

- **根因**：`process.exit()` 在 `console.log` 之后立即调用，pipe 还没 flush。
- **修复**：cleanup 清除 interval 后让进程自然退出，不依赖 process.exit。

### Bug 4：bgTab 元素 "not visible"

- **根因**：bgTab 在 `show: false` 的隐藏窗口中，Playwright 的 actionability check 认为元素不可见。
- **修复**：CDP `Emulation.setDeviceMetricsOverride` + 日期选择器 click 的 DOM evaluate fallback。

### Bug 5：截图超时导致整体失败

- **根因**：`page.screenshot({ fullPage: true })` 等字体加载超时，异常冒泡设置 `finalStatus = "failed"`。
- **修复**：`stepSaveScreenshot` 内部 try/catch，non-fatal。

### Bug 6：SIGKILL 后 bgTab 残留

- **根因**：180s 超时强杀子进程时，`process.once("exit")` 不触发，bgTab 不会被清理。
- **修复**：timeout handler 中写 IPC 文件兜底清理。

### Bug 7：Agent 不传 submitEnabled

- **根因**：Gemini 模型不稳定，有时用 inspect_only 模式。
- **修复**：SKILL.md 加约束，runElectronModeDirect 默认 productType。

## 五、复用的现有机制

| 机制 | 来源 | 说明 |
|------|------|------|
| CDP 端口 9222 | `src/main/index.ts` | Electron 启动时 `--remote-debugging-port=9222` |
| bgTab (BrowserView) | `src/main/window-manager.ts` | `createBgTab()` 创建隐藏 BrowserView |
| IPC 文件通信 | `src/main/window-manager.ts` | 100ms 轮询 `hotel-ai-browser-ipc-*.json` |
| CDP 锁 | `scripts/ai-web-crawler/` | `hotel-ai-browser-cdp.lock` 文件互斥 |
| SkillExecutor | `src/main/skill-executor.ts` | spawn 子进程 + 收集 stdout/stderr |
| Cookie 共享 | Electron session | bgTab 共享主浏览器的 session，自动携带登录态 |
