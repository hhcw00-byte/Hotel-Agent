# 智能调价核心策略

本文档定义智能调价的核心逻辑、决策树和具体规则。

## 核心理念

**以房量（库存）、房价为核心指标，以提价保收益、降价冲流量为基本手段，参考竞品但不盲从。**

## 调价决策树

```
用户调价需求
    ↓
确定时间范围
    ├─ 当天调价 → 实时动态调整（2小时复盘法）
    ├─ 短期调价 → 未来3-7天价格策略
    └─ 长期调价 → 未来1个月以上底价铺设
    ↓
收集三维数据
    ├─ 房态数据（room-status）
    ├─ 本店房价（room-pricing）
    └─ 竞品房价（competitor-pricing）
    ↓
综合分析
    ├─ 供需分析（余房量、预售健康度）
    ├─ 竞争分析（价格定位、竞品差距）
    └─ 收益分析（任务目标、收益优化）
    ↓
生成调价方案
    ├─ 涨价方案（提价保收益）
    ├─ 降价方案（降价冲流量）
    └─ 维持方案（保持现状）
    ↓
输出建议报告
```

## 一、当天调价策略（实时动态调整）

当天调价的核心在于"两小时复盘法"和"余房量阈值判断"。

### 1.1 观察频率

大约每2小时关注一次，重点时段：
- 10:00 - 早上复盘，判断当日需求
- 14:00 - 午后观察，调整策略
- 16:00 - 下午监控，准备晚高峰
- 18:00 - 傍晚评估，决定是否溢价
- 20:00 - 晚间复盘，决定是否甩房

### 1.2 上调逻辑（提价保收益）

**触发条件：**

**条件1：余房量阈值**
- 早上10:00左右，若"团结湖店"（总房量106间）剩余房量少于20间
- 判断：需求强劲，供不应求
- 动作：直接上调价格

**条件2：溢价时段**
- 下午至傍晚，若房量进一步减少
- 判断：供需极度不平衡
- 动作：根据情况挂出高价（溢价销售）

**调价幅度：**
- 常规上调：10-20元
- 溢价上调：20-30元或更高
- 分批上调：先调10元观察反应，若2小时内仍有订单，继续上调

**示例：**
```
时间：10:00
房态：总房量106间，剩余18间
分析：剩余房量<20间阈值，需求强劲
建议：豪华大床房 ¥388 → ¥408 (+¥20)
      标准双床房 ¥288 → ¥308 (+¥20)
理由：供不应求，提价保收益
```

### 1.3 下调逻辑（降价冲流量）

**触发条件：**

**条件1：库存积压**
- 晚上剩余房量远高于预期
- 例如：当晚剩60间，但预估只能卖出30间
- 判断：库存压力大，需要甩房
- 动作：睡前进行大幅度下调

**条件2：无订单观察窗**
- 连续2小时没有任何新订单进账
- 判断：当前价格在市场中失去竞争力
- 动作：执行降价动作

**调价幅度：**
- 常规下调：10-20元
- 甩房下调：20-30元或更高
- 分批下调：先调10元观察反应，若仍无订单，继续下调

**示例：**
```
时间：20:00
房态：总房量106间，剩余65间
分析：剩余房量过多，预估只能卖出30间，库存积压
建议：豪华大床房 ¥388 → ¥358 (-¥30)
      标准双床房 ¥288 → ¥268 (-¥20)
理由：库存压力大，降价冲流量
```

### 1.4 渠道管理

**原则：**
- 保持全平台（携程、美团、飞猪）价格一致
- 避免价格混乱导致客户投诉

**极端情况处理：**
- 可能关闭流量较小的渠道（如飞猪）
- 可能关闭易产生差评的渠道（如靠近马路房型）
- 目的：保护核心渠道的评分

## 二、短期调价策略（未来3-7天）

短期策略侧重于"预售健康度"和"周率差异"。

### 2.1 健康房量梯度

**理想状态：**
店长认为理想的预售状态应呈现阶梯状：
- 明天剩30间
- 后天剩40间
- 大后天剩60间
- 每日相差10间左右为健康

**判断标准：**
```
健康梯度：每日房量差10-15间
预警梯度：每日房量差<5间或>20间
危险梯度：某日房量突然断崖式下降或上升
```

### 2.2 预售速度监控

**监控方法：**
- 提前24小时观察
- 若余房比例超过30%-40%，说明预售不足

**示例：**
```
日期：03-05（明天）
总房量：106间
当前剩余：55间
余房比例：51.9%
判断：预售不足，需提前调整价格引流
建议：下调10-15元刺激预订
```

### 2.3 周中/周末差异

**周五、周六（旺季）：**
- 特点：需求旺盛
- 策略：设定高于平均任务额的价格
- 调价幅度：比平日高20-50元

**周日、周一（淡季）：**
- 特点：需求疲软
- 策略：提前调低价格确保基础稳动率
- 调价幅度：比平日低10-30元

**示例：**
```
周五（03-05）：
  豪华大床房：¥428（比平日+¥40）
  标准双床房：¥328（比平日+¥40）

周一（03-08）：
  豪华大床房：¥368（比平日-¥20）
  标准双床房：¥268（比平日-¥20）
```

### 2.4 任务目标导向

**目标分解：**
- 将月度营业额目标（如30万/月）分解到每天
- 月目标30万 ÷ 30天 = 每天1万元

**反向推算：**
- 根据当日预售与目标的差距，反向推算价格
- 若当日预售仅5000元，距离目标差5000元
- 需要通过提价或增加订单量来弥补差距

**示例：**
```
月目标：30万
日目标：1万
当日预售：6000元
差距：4000元
剩余房量：30间
需要：平均每间房多赚133元，或多卖10间房
建议：适度提价10-20元，同时保持订单量
```

## 三、长期调价策略（未来1个月）

长期策略侧重于"底价铺设"与"异动监控"。

### 3.1 远期铺价

**时间要求：**
- 至少提前3到6个月在系统铺设价格
- 越早铺设，越能抓住早鸟客户

**铺价原则：**
- 远期价格通常设置得较高
- 防止由于未发现的突发热点（展会、演唱会等）被人以极低价格"捡漏"锁定

**示例：**
```
当前日期：2026-03-03
铺设日期：2026-06-03（3个月后）
铺设价格：
  豪华大床房：¥480（防守型高价）
  标准双床房：¥380（防守型高价）
理由：防止热点被捡漏，后续可根据实际情况下调
```

### 3.2 热点异动预警

**异动点识别：**
- 若未来2周甚至更远日期，某天的订单量突然超过总量的20%
- 说明该日期有潜在热点（展会、演唱会、大型活动等）

**处理动作：**
- 立即根据该热点信息大幅度提价
- 提价幅度：50-100元或更高
- 分批次拉升：根据订单进场速度持续提价

**示例：**
```
异常日期：2026-04-15
订单量：已预订25间（占总量23.6%）
距离日期：43天
判断：该日期有潜在热点
建议：立即提价
  豪华大床房：¥480 → ¥580 (+¥100)
  标准双床房：¥380 → ¥460 (+¥80)
后续：每进10单，再提价20元
```

### 3.3 大促与任务平衡

**长假策略（国庆、春节等）：**
- 提价动作要早：至少提前2-3个月开始提价
- 分批次拉升：根据订单进场速度分批次提价
- 不要一次性拉满：避免吓跑客户

**提价节奏：**
```
提前3个月：基础价 + ¥50
提前2个月：基础价 + ¥80
提前1个月：基础价 + ¥120
提前2周：基础价 + ¥150
提前1周：基础价 + ¥180
根据订单速度：持续微调
```

**示例：**
```
目标日期：2026-10-01（国庆）
当前日期：2026-07-01（提前3个月）
当前价格：¥388
建议价格：¥438（+¥50）
后续策略：每月根据订单速度提价30-40元
预期最终价格：¥568（+¥180）
```

## 四、竞争对手参考策略

### 4.1 核心竞品

**主要参考对象：**
- 孙团结湖店（首要参考）
- 西雅图三元桥店

### 4.2 参考逻辑

**原则：仅参考，不盲从**

**理由：**
- 竞品店长也可能存在误判
- 如：深夜突然降价甩房
- 或：定价过高导致无人问津

**参考时机：**
- 仅在自身房量异常时对标竞品
- 房量过多或过少（异常值）时，才去对标竞品价格

### 4.3 引流价策略

**库存压力大时：**
- 价格需低于竞品以截流
- 目标：吸引价格敏感客户
- 差价：比竞品低10-30元

**库存压力小时：**
- 价格可高于竞品以获取更高收益
- 目标：提升客单价
- 差价：比竞品高10-30元

**示例：**
```
场景1：库存压力大
竞品价格：¥420
本店价格：¥390（-¥30）
理由：库存积压，需要截流

场景2：库存压力小
竞品价格：¥420
本店价格：¥450（+¥30）
理由：供不应求，提价保收益
```

## 五、调价参考因素权重

### 5.1 直接影响价格（权重：高）

**房态（最重要）：**
- 当前房态：实时入住情况
- 近期房态：未来3-7天预订情况
- 房态日历：未来1个月房态信息
- 远期预定：未来的房量预定

**权重分配：**
- 当前房态：40%
- 近期房态：30%
- 远期预定：20%
- 其他因素：10%

### 5.2 调整价格参考（权重：中）

**竞品价格：**
- 仅在房量异常时参考
- 权重：20%

**平台预定量占比：**
- 美团、携程、飞猪的订单占比
- 权重：15%

**任务目标（业绩）：**
- 运营方下发的月目标
- 权重：15%

### 5.3 调价区间

**常规调整：**
- 一般上下调整10-30元
- 小幅调整：10-15元
- 中幅调整：15-25元
- 大幅调整：25-30元

**房型调整：**
- 正常情况：根据单人间和多人间两部分房型做价格调整
- 特殊情况：针对单个房型（如静谧单人间空房较多）调整指定房型价格

## 七、风险控制

### 7.1 涨价风险

**风险：**
- 价格过高导致订单量下降
- 客户流失到竞品

**控制措施：**
- 分批次上调，观察反应
- 2小时无订单立即调整
- 保持与竞品合理价差

### 7.2 降价风险

**风险：**
- 价格过低影响收益
- 低于成本价亏损
- 频繁降价影响品牌定位

**控制措施：**
- 设定价格底线（成本价+合理利润）
- 降价前评估收益影响
- 避免频繁大幅降价

### 7.3 渠道风险

**风险：**
- 不同平台价格不一致导致投诉
- 某渠道差评影响整体评分

**控制措施：**
- 保持全平台价格一致
- 极端情况可关闭问题渠道
- 重点保护核心渠道评分

## 九、常见场景处理

### 场景1：周末房量紧张

```
数据：周六，剩余房量15间，竞品¥450
分析：供不应求，周末旺季
决策：上调价格
建议：¥388 → ¥438 (+¥50)
理由：需求强劲，可溢价销售
```

### 场景2：平日库存积压

```
数据：周二，剩余房量70间，竞品¥380
分析：库存压力大，需要引流
决策：下调价格
建议：¥388 → ¥358 (-¥30)
理由：库存积压，降价冲流量
```

### 场景3：热点异动

```
数据：未来某日订单量突增至30间（28%）
分析：潜在热点（展会/演唱会）
决策：立即大幅提价
建议：¥388 → ¥488 (+¥100)
理由：热点机会，抓住溢价空间
```

### 场景4：连续无订单

```
数据：连续2小时无新订单，剩余40间
分析：价格失去竞争力
决策：降价引流
建议：¥388 → ¥368 (-¥20)
理由：需要刺激订单，避免库存积压
```

## 十、总结

智能调价的本质是**动态平衡供需关系，在入住率和房价之间找到最优解**。

**核心要点：**
1. 以房态为第一决策依据
2. 参考竞品但不盲从
3. 分批次调整，快速响应
4. 提前布局，抓住热点
5. 持续监控，灵活调整
