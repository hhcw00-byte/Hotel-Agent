# 智能回复助手

本策略适用于用户已在浏览器中打开目标页面（评价管理页、IM消息页等）的场景，直接提取当前页面内容并生成回复参考，**无需爬虫自行导航**。

## 工作流程

### Step 1: 提取当前页面内容

调用 `ai_web_crawler` 的 `extract_current` 操作，直接读取用户当前打开的页面：

```json
{
  "operation": "extract_current",
  "extraction_goal": "提取页面中所有可见的评价或消息内容，包括：用户名/昵称、评分（如有）、评价/消息正文、时间、房型（如有）、回复状态（已回复/待回复）"
}
```

> **注意**：`extract_current` 不会切换标签页，也不会导航，直接提取 Electron 浏览器当前激活页面的内容。

### Step 2: 判断页面类型

根据提取到的内容，判断当前页面属于哪种场景：

- **评价页**：包含评分（星级）、入住日期、房型等字段 → 走评价回复流程
- **IM 消息页**：包含聊天气泡、发送时间、消息状态等字段 → 走 IM 回复流程
- **非目标页面**：如果提取到的内容中没有评价/消息相关信息（如当前在首页、房价管理页、订单页等），**必须立即告知用户**：

```
当前页面不是评价/消息页面，无法提取评价内容。
请在浏览器中切换到以下页面之一：
- 携程 ebooking → 点评管理
- 美团商家后台 → 评价管理
- Booking 后台 → 住客评语
- 任意 OTA 的 IM/消息页面

切换后再告诉我，我会重新提取内容。
```

**不要在非目标页面时尝试导航或猜测内容，直接提示用户切换页面。**

### Step 3: 分析内容

**评价场景分析：**
- 情感判断：正面（评分高、内容积极）/ 负面（评分低、包含投诉）/ 中性
- 提取称赞点、投诉点、改进建议
- 优先处理未回复的评价

**IM 消息场景分析：**
- 消息分类：咨询类 / 投诉类 / 预订类 / 其他
- 提取消息正文、发送时间、客户信息
- 优先处理待回复的消息

### Step 4: 生成回复参考

根据内容类型生成对应回复，**回复参考必须使用 Markdown 引用格式（以 `>` 开头）**，前端才能正确渲染复制按钮。

## 输出格式

### 评价回复格式

```markdown
## 评价回复参考

### 评价 1
**用户：** 张先生 | **评分：** 4.8/5 | **房型：** 豪华大床房
**原始评价：** 房间很干净，服务态度很好...
**情感：** 正面
**回复参考：**
> 尊敬的张先生，感谢您的好评与认可！很高兴我们的房间卫生和服务态度得到了您的肯定。期待您的再次光临！

### 评价 2
**用户：** 李女士 | **评分：** 2.5/5 | **房型：** 标准双床房
**原始评价：** 隔音效果太差，晚上吵得睡不着...
**情感：** 负面
**回复参考：**
> 尊敬的李女士，非常抱歉给您带来不好的入住体验。关于隔音问题，我们已记录并将持续改善。如您方便，希望能为您提供适当补偿。再次深表歉意！
```

### IM 消息回复格式

```markdown
## IM 消息回复参考

### 消息 1
**客户：** 王先生 | **时间：** 14:30 | **状态：** 待回复
**原始消息：** 你好，请问有停车场吗？收费吗？
**消息类型：** 咨询类
**回复参考：**
> 您好王先生！我们酒店配有地下停车场，提供24小时免费停车服务，入口在酒店正门右侧。如需帮助请联系前台，期待您的光临！
```

## 回复质量要求

**评价回复：**
- 长度控制在 50-200 字
- 语气专业、真诚、个性化，避免千篇一律
- 负面评价要具体回应问题，不要泛泛而谈
- 如评价中提到具体员工，可在回复中提及表扬

**IM 消息回复：**
- 咨询类：提供准确信息 + 引导预订
- 投诉类：致歉 + 解决方案 + 补偿意愿
- 预订类：确认信息 + 操作指引
- 语气热情、专业、及时

## 注意事项

- 不要编造任何评价内容或数据，只基于提取到的真实内容生成回复
- 如果页面内容为空或无法识别，告知用户并建议手动导航到目标页面
- 最终回复仅供参考，由酒店运营/客服人员审核后使用
- 如果用户明确指定了筛选条件（如"只看差评"、"未回复的"），优先处理对应内容
